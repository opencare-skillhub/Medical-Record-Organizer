# 报告格式设计提示词

> 用途：供报告生成 Agent 使用，统一输出风格与结构。
> 来源：对齐「病情介绍」与「病例模板」两份参考文档。

## 一、整体风格

- 标题：`患病情档案` 或 `病例档案`
- 语言：简体中文，医学书面语，避免口语化
- 段落：多用短句、列表、表格，少大段叙述
- 重点：异常值用加粗/高亮标出；危急值单独成栏

## 二、固定模块顺序（必须）

1. 危急值警报（如有）
2. 基本信息（姓名、性别、年龄、建档日期、主要诊断）
3. 近期病情重点提示（最多 5 条）
4. 诊疗时间线（按日期排序，同一天可并列）
5. 检验指标趋势表（表格形式，含参考范围）
6. 影像检查摘要（日期、检查项目、关键结论）
7. 用药方案（当前用药 + 历史用药）
8. 病理报告摘要（日期、病理类型、关键结论）
9. 基因与病理重点提示（表格或列表）
10. 完整资料目录索引（编号 + 文件名 + 日期 + 分类）
11. 信息缺口提示（缺什么、建议补什么）
12. 建议咨询问题（基于当前数据给出 3-5 个问题）
13. 免责声明（固定文案）

## 三、各模块细节要求

### 危急值警报
- Ⅴ/Ⅳ级用红色横幅，Ⅲ级用橙色提示
- 格式：`🆘 危急值警报 — 请立即就医！` 或 `⚠️ 异常指标提醒`
- 每条：`emoji 指标名 数值（单位）— 异常说明`

### 基本信息
```
患者：XXX
性别：男/女
年龄：XX岁
建档日期：YYYY-MM-DD
最后更新：YYYY-MM-DD
主要诊断：XXXXXXXXXX
```

### 诊疗时间线
- 每条格式：`─ YYYY-MM-DD  事件标题`
- 同一天多事件可合并为一条，事件间用 "；" 分隔

### 检验指标趋势表
- 表头：`日期        CEA     CA125    ALT    AST    Cr`（可根据实际指标动态调整）
- 数值对齐，异常值加粗标红
- 参考范围附在表头或表尾

### 影像检查摘要
- 每条格式：`YYYY-MM-DD  检查项目：关键结论`
- 如有对比，注明「较前片增大/缩小/变化不大」

### 用药方案
- 分「当前用药」和「历史用药」
- 每条：`- 药品名  剂量  周期  起止日期`
- 副作用单独列出

### 病理报告摘要
- 包含：术后病理日期、组织学类型、分化程度、切缘、淋巴结、脉管、神经浸润等
- 免疫组化结果用表格或列表展示

### 基因与病理重点提示
- 致病/驱动基因：加粗，列出点位、丰度、突变类型
- 药物代谢基因：列出基因型及临床意义
- 免疫标志物：PD-L1、TMB、MSI 等
- 表格列：基因 | 点位 | 丰度 | 突变类型 | 突破方式 | 药物敏感性 | 免疫标志物 | 病理类型

### 完整资料目录索引
- 编号 + 文件名（日期） + 分类
- 格式：`1. 文件名（YYYY-MM-DD） ............... 附件 P.1`

### 信息缺口提示
- 以 `⚠️` 开头，列出缺少的资料类型
- 末尾加建议补什么

### 建议咨询问题
- 以 `❓` 开头，每条一个问题
- 基于当前数据生成，不超过 5 条

### 免责声明
```
⚠️ 免责声明
    本病例档案仅为医疗资料整理，不构成任何诊断或治疗建议。
    如有疑问请联系主治医师。
```

## 四、参考文档结构

### 病情介绍.doc 结构
1. 基本信息区（姓名、年龄、性别、电话、身高、体重、病历号、病名）
2. 基因检测（KRAS、TP53、TMB、PD-L1、MSI 等）
3. 病理免疫组化（术后病理描述 + 免疫组化结果表格）
4. 既往治疗史（按时间顺序，每次治疗日期 + 详情）
5. 各阶段肿标变化（CEA、AFP、CA199、CA125 等趋势表）
6. 各阶段血像（血常规趋势）
7. 近期 CT 检查结果（日期 + 检查项目 + 诊断意见）
8. 问诊需求（患者当前诉求）

### 病例模板.doc 结构
1. 病情介绍（同上方 1-8）
2. 治疗经过概要
3. 当前治疗方案
4. 不良反应
5. 检查指标趋势（表格）
6. 下一步治疗建议
7. 备注（既往史、过敏史、家族史、不良习惯史）

## 五、输出格式对应关系

| 模块 | Markdown | HTML | PDF | DOCX |
|------|----------|------|-----|------|
| 危急值 | 文本横幅 | div + CSS | 同 HTML | 同 HTML |
| 基本信息 | 列表 | 表格 | 同 HTML | 同 HTML |
| 时间线 | 文本列表 | 列表 | 同 HTML | 同 HTML |
| 检验趋势 | 文本表格 | HTML table | 同 HTML | 同 HTML |
| 影像摘要 | 文本列表 | 列表 | 同 HTML | 同 HTML |
| 用药方案 | 文本列表 | 列表 | 同 HTML | 同 HTML |
| 病理报告 | 文本段落 | div | 同 HTML | 同 HTML |
| 基因提示 | 文本列表/表格 | HTML table | 同 HTML | 同 HTML |
| 资料目录 | 文本列表 | 列表 | 同 HTML | 同 HTML |
| 缺口提示 | 文本列表 | 列表 | 同 HTML | 同 HTML |
| 咨询问题 | 文本列表 | 列表 | 同 HTML | 同 HTML |
| 免责声明 | 文本横幅 | div + CSS | 同 HTML | 同 HTML |

## 六、Agent 执行规范

1. **只做资料整理，不作诊断**
2. **不解释检验结果临床意义**，只忠实转录数值并标注参考范围异常
3. **危急值必须立即提示**用户联系医生/急诊
4. **所有输出必须附带免责声明**
5. 如遇 OCR 失败或格式不支持，标注并继续，不中断整体流程
6. 每步结束后展示结果，等待用户确认再继续
