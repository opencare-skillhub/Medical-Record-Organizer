# HTML 报告优化设计文档

> 版本：v1.0  
> 创建日期：2026-06-23  
> 用途：定义 HTML 报告的视觉设计、交互行为和折叠详情规范  
> 对照基准：用户提供的两份报告截图（患者基础信息 + CA199 趋势）

---

## 1. 设计目标

1. **概要优先**：医生打开报告 10 秒内看到关键信息
2. **详情可折叠**：点击展开查看完整数据，不干扰主视图
3. **视觉分区明确**：不同模块用不同颜色区分
4. **移动端友好**：响应式布局，手机/平板可读
5. **打印友好**：折叠区域在打印时自动展开

## 3. 组件设计规范

### 3.1 卡片组件（Card）

**用途**：患者基础信息、基因检测结果、关注问题要点

**样式**：
- 白色背景，圆角 8px
- 顶部彩色标题栏（4px 高度）
- 内边距 16px
- 阴影：0 2px 8px rgba(0,0,0,0.08)

**颜色方案**：
| 模块 | 标题颜色 | 图标 |
|------|---------|------|
| 患者基础信息 | #2563eb (蓝) | 👤 |
| 手术病理结果 | #1e40af (深蓝) | 🔬 |
| 免疫组化 & PD-L1 | #059669 (绿) | 🧪 |
| 基因检测结果 | #d97706 (橙) | 🧬 |
| 关注问题要点 | #dc2626 (红) | ⚠️ |
| 用药方案 | #7c3aed (紫) | 💊 |
| 影像检查 | #0891b2 (青) | 🏥 |

### 3.2 时间轴组件（Timeline）

**用途**：就诊经历时间线

**样式**：
- 垂直时间线，左侧绿色圆点
- 每条记录：日期 + 事件 + 关键结果
- 疗效评估用颜色标签：CR(绿)/PR(蓝)/SD(黄)/PD(红)

**HTML 结构**：
```html
<div class="timeline">
  <div class="timeline-item">
    <div class="timeline-marker"></div>
    <div class="timeline-content">
      <div class="timeline-date">2026-01-20</div>
      <div class="timeline-title">皮肤发现黄斑</div>
      <div class="timeline-result">检查显示胰头占位</div>
    </div>
  </div>
</div>
```

### 3.3 表格组件（Table）

**用途**：病理诊断、免疫组化、基因检测、肿瘤标志物

**样式**：
- 表头：彩色背景 + 白色文字
- 斑马纹：偶数行浅灰背景
- 悬停高亮
- 响应式：小屏幕横向滚动

**特殊表格：肿瘤标志物趋势表**
- 表头：参考范围（灰色小字）
- 异常值：红色加粗
- 正常值：黑色
- 趋势箭头：↑ 升高 / ↓ 降低 / → 持平

### 3.4 图表组件（Chart）

**用途**：CA199 等指标趋势

**实现**：使用 Chart.js 或纯 SVG

**样式**：
- 折线图，平滑曲线
- 数据点：圆形标记
- 参考线：虚线表示正常范围
- 悬停提示：显示日期和数值

**折叠行为**：
- 默认显示图表
- 点击"查看完整数据"展开数据表格

### 3.5 折叠组件（Collapsible）

**用途**：详情文本、原始数据、附件列表

**触发样式**：
- 标题行：浅灰背景，右侧箭头 ▼
- 点击展开/收起
- 过渡动画：0.3s ease

**HTML 结构**：
```html
<div class="collapsible">
  <div class="collapsible-header" onclick="toggle(this)">
    <span>📋 完整病理描述</span>
    <span class="arrow">▼</span>
  </div>
  <div class="collapsible-body">
    <!-- 详细内容 -->
  </div>
</div>
```

## 5. 交互设计

### 5.1 折叠/展开交互

**触发方式**：
- 点击标题行展开/收起
- 键盘可访问（Tab + Enter）
- 移动端触摸友好

**动画**：
```css
.collapsible-body {
  max-height: 0;
  overflow: hidden;
  transition: max-height 0.3s ease, padding 0.3s ease;
}
.collapsible.open .collapsible-body {
  max-height: 5000px;
  padding: 16px;
}
```

**状态指示**：
- 收起：箭头 ▼
- 展开：箭头 ▲
- 悬停：背景色变深

### 5.2 打印优化

```css
@media print {
  .collapsible-body {
    max-height: none !important;
    display: block !important;
  }
  .collapsible-arrow {
    display: none;
  }
}
```

### 5.3 响应式断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| Desktop | >1024px | 多列网格 |
| Tablet | 768-1024px | 单列，最大宽度 800px |
| Mobile | <768px | 单列，全宽，字体缩小 |

## 8. 实现规范

### 8.1 技术栈

- HTML5 + CSS3（无外部依赖）
- Vanilla JavaScript（折叠交互）
- Chart.js（趋势图表，可选）
- 响应式设计（CSS Grid + Flexbox）

### 8.2 文件结构

```
output/
├── case_report.html      # 主报告（单文件，内嵌 CSS/JS）
├── case_report.md        # Markdown 源文件
└── assets/
    ├── chart.js          # 图表库（如需要）
    └── styles.css        # 独立样式（可选）
```

### 8.3 性能要求

- 单文件大小 < 500KB
- 首屏渲染 < 1s
- 折叠动画 60fps
- 移动端触摸响应 < 100ms

### 12.1 核心设计点（折叠策略）

**「保留概要 + 详情可折叠」三段式结构**，每个模块包含：

1. **概要（Summary）**：3-5 条关键信息，默认可见 → 医生 1 分钟掌握
2. **详情（Detail）**：完整原始数据，折叠保留 → 不丢失任何信息
3. **触发器（Trigger）**：`展开详情 ▾` 原生 `<details>` 元素，点击展开

**卡片本身不折叠**，永远显示概要；折叠只发生在详情子区域，用原生 `<details>` 实现（无需 JS）。

```html
<section class="card blue">                    <!-- 卡片：全边框+浅底色，概要常驻 -->
  <h2>💊 用药方案</h2>
  <div>当前用药：AG方案 38次</div>              <!-- 概要 -->
  <div class="detail">                          <!-- 折叠详情 -->
    <details><summary>展开完整详情 ▾</summary>
    <div class="detail-body">原始处方全文…</div>  <!-- 详情保留 -->
    </details>
  </div>
</section>
```

### 12.2 卡片风格要点（对齐参考模板）

参考 `pancreatic_cancer_case_template.html` 的视觉风格：

| 设计点 | 实现 |
|--------|------|

### 12.3 render_report.py 集成方案

当前 `render_html()` 通过 Markdown → HTML 转换生成报告。新方案改为**直接渲染 Jinja2 HTML 模板**：

```python
def render_html(manifest, *, timeline=None, output_path=None, report_context=None):
    env = Environment(loader=FileSystemLoader(str(TEMPLATE_DIR)), autoescape=True)
    env.globals.update(CSS=_HTML_CSS, JS=_HTML_JS)  # 内嵌样式和脚本
    template = env.get_template('html-report-template.html')
    ctx = report_context or compute_report_context(manifest, timeline=timeline)
    html = template.render(**ctx)
    output_path.write_text(html, encoding='utf-8')
    return output_path
```

**渐进迁移策略**：
- 保留现有 Markdown 通道（`render_md`）不变
- 新增 `render_html` 走 Jinja2 模板分支
- 通过配置项切换：`output_format='md' | 'html' | 'html_card'`
